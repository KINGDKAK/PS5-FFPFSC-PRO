  v1.3.0  (current)

NEW FEATURES

  COMMUNITY COMPATIBILITY
  • Live Google Sheet integration — fetches 5,000+ community game
    reports directly inside the app (Compatibility tab)
  • Search bar + status filter (Working / Partial / Not Working /
    Not Tested) on the compatibility list
  • Vote-based submission — one bad report cannot override the
    community majority
  • Duplicate submit protection — resubmitting the same game
    replaces the existing entry instead of stacking rows

  WHAT'S NEW DIALOG + FEATURE TOUR
  • First-run dialog after every update shows all new features with
    descriptions — skippable, re-launchable at any time
  • "Take a Feature Tour" — interactive arrow callouts pointing to
    new UI elements one at a time
  • Re-launchable from Settings → About and the top of Help / FAQ
    (buttons always visible in a fixed header, not buried in scroll)

  APR / AMPR GAME SUPPORT (fully overhauled)
  • Checkbox removed — detection and setup are now fully automatic
  • Auto-detects APR titles via playgo-chunk.dat in sce_sys/
  • Before compression, if not auto-detected, app asks:
    "Is this an APR / AMPR title?" with a clear list of what will
    happen if Yes is selected
  • If AMPR emu folder is not set in Settings, a second dialog
    prompts the user to pick the folder right then (Browse button
    included) — saves the path to Settings automatically
  • fakelib/ folder created inside the game directory and
    libSceAmpr.sprx + libScePlayGo.sprx injected before compression
  • AMPR index (ampr_emu.index) built automatically after injection
    using the AMPRIDX3 binary format with FNV1a64 hashing and an
    open-addressed hash table — covers all /app0 paths including
    the injected fakelib files
  • APR handling runs in both single-game and batch auto-advance
    mode — every game in a queue gets the same prompts

  HELP / FAQ TAB
  • Full FAQ covering common compression errors, OOM, AMPR setup,
    ShadowMount mounting, and archive input
  • YouTube tutorial link (KINGDKAK channel)
  • Tour buttons pinned to a fixed header at the top — always
    visible regardless of scroll position

  SETTINGS
  • What's New and Take a Feature Tour buttons in Settings → About
  • AMPR Emu Folder setting added to Folders section

  COMPRESSION PRESETS
  • One-click profiles: Fast / Balanced / Max Compression / Low RAM
    — sets compression level, CPU cores, and block size together

  LIVE RAM METER
  • Available RAM shown in real time in the log header
  • Updates every 2 seconds during compression

  AUTO UPDATE CHECK
  • Silently checks for a new version 3 seconds after the app opens
  • Only notifies if a newer version is available

  OTHER
  • Windows toast notification when compression finishes
  • Auto-retry on out-of-memory — drops CPU count by 1 and retries
    the failed game automatically
  • Block size selector: auto, auto-fit, 16384, 32768, 65536
  • Per-game output subfolder option (output/GameName/)
  • Multi-image queue — drag in multiple .exfat / .ffpkg images
  • Compression ratio in completion log (original → compressed size
    and exact % saved)

FIXED

  PROGRESS BAR
  • Overall % frozen at 97% for large games — root cause: stage
    order in _STAGE_ORDER was wrong (Verify was after Compressing
    and Writing, so the no-backwards guard blocked all transitions
    after verify finished)
  • Correct pipeline order now enforced:
      Scanning → Reading → Creating Temp PFS → Verifying Output
      → Compressing (outer MkPFS) → Writing Final Image → Cleaning Up
  • Outer MkPFS compression now receives 40% of the overall bar
    (was 0% — it was undetectable, progress sat at 97% the whole time)
  • "outer container" and MkPFS log lines now correctly map to
    Compressing stage instead of Writing Final Image
  • Stage weights rebalanced for accurate ETA on large games:
      Scanning 0–5%  |  Reading 5–15%  |  Temp PFS 15–38%
      Verify 38–48%  |  Compressing 48–88%  |  Writing 88–97%
      Cleaning Up 97–100%

  APR / AMPR STABILITY
  • [ERROR] _poll crash — GameItem objects created via from_exfat()
    or reconstructed from session history were missing the ampr_emu
    attribute; added class-level default (ampr_emu = False) and
    changed all read sites to use getattr() defensively
  • APR ask dialog now fires in batch auto-advance mode — previously
    _batch_auto_start() called CLIWorker directly, bypassing start()
    and the APR check entirely

  OTHER
  • Python < 3.10 crash (str | None union type hint syntax)
  • Unicode arrow crash on cp1252 Windows consoles
  • Multiple game folders false positive — nested sce_sys / eboot.bin
    no longer counted as a second game
  • App crash on startup when a saved temp/output folder path no
    longer exists on disk
  • OOM error message now correctly lists "Uncheck Verify Output"
    as step 1
  • CPU cores slider uncapped — now goes up to 16 cores

────────────────────────────────────────────────────────────────────────
  v1.2.2
────────────────────────────────────────────────────────────────────────

STAGE DISPLAY OVERHAUL
  • Fixed stages permanently stuck on "Scanning Files" — early
    backend path strings were triggering "Creating Temp PFS" before
    scanning began, locking the no-backwards guard
  • Fixed "write" progress bars not advancing Temp PFS % — backend
    labels include trailing speed/ETA; switched to first-word matching
  • Fixed "Verifying Output" firing at startup from the MkPFS
    "post-build verify is disabled" info message
  • Added auto-advance: Compressing 100% now pushes display forward
    to "Writing Final Image" automatically
  • Stage cascade snap — when a later stage begins, all earlier
    stages are forced to 100% so breadcrumbs never show stale %

CRITICAL BUG FIX — stdout / process handling
  • proc.wait() was inside the stdout reading loop — blocked on the
    first line until the process finished; fixed to consume all
    stdout before calling wait()

RAW LOG FIXES
  • Log file flushed to disk every 30 s during the run (previously
    only written at job completion)
  • Log flood fixed — progress entries only written at 5% bucket
    boundaries and at 100%

ARCHIVE EXTRACTION
  • RAR extraction fixed — 7-Zip detected at default install path
    without needing it on PATH
  • _find_root improved — BFS search locates game folder via
    sce_sys/param.json inside extracted content

GAME NAME FIX
  • Games showing as parent folder name fixed
  • guess_game_name() priority: (1) param.json localised title,
    (2) strip -app/_app suffix, (3) never use parent folder name

COMMUNITY COMPATIBILITY
  • Duplicate reports fixed — resubmitting replaces existing entry
  • Startup re-test reminder 4 s after launch for untested games
